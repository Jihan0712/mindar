Admin Account Setup

Admin accounts require server-side provisioning

First admin — bootstrap API

The very first admin account is created via a one-time API call using a secret key set in your Cloudflare Worker environment (BOOTSTRAP_ADMIN_KEY).

Show bootstrap command ›
# 1. Set BOOTSTRAP_ADMIN_KEY as a Worker secret in Cloudflare Dashboard
# 2. Run this once to create the first admin account:
 
curl -X POST https://<your-worker>.workers.dev/api/auth/bootstrap-admin \
  -H "Content-Type: application/json" \
  -H "x-bootstrap-key: YOUR_BOOTSTRAP_KEY" \
  -d '{"email":"admin@inrl.com","password":"strong-password"}'

⚠ This endpoint only works once per email. Rotate or delete BOOTSTRAP_ADMIN_KEY from your Worker secrets after use.

Additional admins

After the first admin is set up, additional admin accounts are created by inserting a row directly into the D1 users table with role = 'admin', or by running a migration script.
